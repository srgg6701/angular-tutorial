Создание каркаса сайта

  1. Создать основной файл (точку входа). Как правило, это — index.html
  2. Создать внутри файла-точки входа блок для загрузки контента:
    3. <ng-view></ng-view>
  3. Создать директорию для шаблонов (например, — "templates") и файлы для контента соответствующих разделов.
  4. Создать контроллеры для шаблонов.
  5. Настроить роутинг. На этом моменте необходимо остановиться подробнее. Вот пример файла с роутингом: 
    'use strict';
var app = angular.module('tApp',[])
  .config( function($routeProvider, $locationProvider){
    $routeProvider
        .when('/',{
            templateUrl:'templates/default.html',
            controller: 'defaultController'
        })
        .when('/skeleton',{
            templateUrl:'templates/skeleton.html',
            controller: 'skeletonController'
        })
        .when('/contacts',{
            templateUrl:'templates/contacts.html',
            controller: 'contactsController'
        })
        .when('/test',{
            templateUrl:'templates/test.html',
            controller: 'testController'
        })
        .otherwise({
            templateUrl:'templates/404.html',
            controller: 'defaultController'
        });
    $locationProvider.html5Mode(true);
});

    Как можно заметить, данный файл является основным модулем приложения, а роутинг задаётся в его конфигурации. Чтобы всё это работало, следует передать модулю встроенный angular-сервис $routeProvider, являющийся компонентом родительского сервиса $route.

    Обратите внимание, что здесь задействован ещё один встроенный сервис — $locationProvider, в конце конфигуратора. Если

    Разбор URL осуществляется методом .when(), который определяет, какие шаблон и контроллер соответствуют разделу, загружаемому по текущему адресу. Для того, чтобы обрабатывать неизвестные роутеру URL, используйте метод .otherwise. В частности, таким образом можно перенаправлять пользователя на страницу 404.
